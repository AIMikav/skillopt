—— THE PROBLEM

Agent Skills Decay Over Time

  • Filler phrases accumulate — “make sure to”, “ensure that”, “don’t forget to” add tokens without adding value
  • Verbose by default — explaining concepts the model already knows wastes context window
  • Redundant patterns — the same command repeated with minor variations instead of consolidated
  • No verification — after trimming, how do you know the skill still handles every use case?
Make skills smaller without breaking them — and prove they still work.
SkillOpt couples optimization with validation. Every rewrite is tested against eval assertions before it can replace the original.

—— EVAL-DRIVEN

Test Cases Ground the Optimization

What You Provide

Realistic user prompts that exercise different parts of the skill. What would someone actually ask?
"My pod shows CrashLoopBackOff. Help me figure out what's wrong."

What Gets Generated

Objective, verifiable assertions the optimized skill must still enable.
  • Includes kubectl logs with --previous
  • Explains common CrashLoop causes
  • Suggests checking container-level logs
No evals? Use --generate-evals — the tool creates test cases and assertions from the skill content itself.
—— HOW IT SCORES

Two Lenses, One Score

Static Analysis

Is the skill well-written?
  • Filler phrases removed
  • Concise without over-trimming
  • Code blocks preserved
  • Structure intact (frontmatter, sections)

LLM-as-Judge

Does the skill still work?
  • Each assertion evaluated independently
  • Strict: must contain actionable content
  • Returns PASS or FAIL with evidence
  • Prevents optimization from gutting functionality
Neither lens alone is enough. Together they ensure the result is both concise and correct.
—— DEMO

Running SkillOpt
